<div class="container">

  <h1>
    CollectionView
  </h1>

  <p>
    The 
    <a href="https://wijmo.com/5/docs/topic/wijmo.odata.ODataCollectionView.Class.html" target="_blank">CollectionView</a>
    class is the core data provider for all Wijmo controls.
    It takes a <b>sourceCollection</b> array and provides 
    data views with sorting, filtering, grouping, paging, 
    notifications, change tracking, currency management,
    and more.</p>
  <p>
    The views created by the <b>CollectionView</b> are exposed
    through the <b>items</b> property, which contains an
    array with sorted/filtered/grouped/paged items from the
    <b>sourceCollection</b> array.</p>
  <p>
    Wijmo controls that have an <b>itemsSource</b> property 
    accept <b>CollectionView</b> objects as well as regular 
    arrays. If regular arrays are used, the control automatically
    creates an internal <b>CollectionView</b> to wrap the original
    data. This internal <b>CollectionView</b> is exposed by the
    control's <b>collectionView</b> property.</p>
  <p>
    The <b>CollectionView</b> class provides extensive functionality
    and a consistent way for controls to deal with data.</p>
  <p>
    Wijmo includes classes that extend <b>CollectionView</b> to
    provide additional services. The best example is
    <a href="https://wijmo.com/5/docs/topic/wijmo.odata.ODataCollectionView.Class.html" target="_blank">ODataCollectionView</a>,
    which provides access to 
    <a href="https://www.odata.org/" target="_blank">OData</a> sources.
  </p>
</div>
