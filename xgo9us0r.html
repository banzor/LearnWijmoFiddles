  </h1>
  <p>
    The <b>CollectionView</b> class relies on a <b>sourceCollection</b>
    that contains an array of data items. This collection is typically
    populated with http requests that retrieve data from a server.</p>
  <p>
    In some cases, it may be advantageous to extend the <b>CollectionView</b> 
    class in order to take advantage of functionality exposed by the
    server. For example, Wijmo includes a <b>ODataCollectionView</b>
    that takes advantage of the <b>OData</b> protocol to provide 
    sorting, filtering, and paging on the server.</p>
  <p>
    Here are some links to samples and demos that show how you can
    extend the <b>CollectionView</b> class to deal with some popular
    data providers:</p>

  <ol>
    <li>
      <a href="https://demos.wijmo.com/5/SampleExplorer/SampleExplorer/Sample/OData" target="_blank">
        OData
      </a>: 
      Shows how to use the <b>wijmo.odata.ODataCollectionView</b> class
      to connect to OData services.
    </li>
    <li>
      <a href="https://demos.wijmo.com/5/SampleExplorer/SampleExplorer/Sample/VirtualData" target="_blank">
        Virtual OData
      </a>: 
      Shows how to use the <b>wijmo.odata.ODataVirtualCollectionView</b> class
      to connect to OData services and show data on-demand.
    </li>
    <li>
      <a href="https://demos.wijmo.com/5/SampleExplorer/SampleExplorer/Sample/Breeze" target="_blank">
        Breeze
      </a>: 
      Shows how to implement a <b>CollectionView</b> class that works with 
      the BreezeJS data library.
    </li>
    <li>
      <a href="https://demos.wijmo.com/5/SampleExplorer/SampleExplorer/Sample/FireBase" target="_blank">
        Firebase
      </a>:

      Firebase databases.
    </li>
