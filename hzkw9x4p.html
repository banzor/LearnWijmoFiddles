<div class="container">

  <h1>
    Grid Events
  </h1>
  <p>
    The FlexGrid provides many events you can use to 
    customize almost every aspect of the control's 
    behavior. You can see a complete list in the
    <a href="https://wijmo.com/5/docs/topic/wijmo.grid.FlexGrid.Class.html" target="_blank">Online Documentation</a>.</p>
  <p>
    These events can be groped into categories that include:</p>
  <ul>
    <li>
      <b>Mouse</b>:
      The FlexGrid relies on HTML events for mouse handling,
      and includes a <b>hitTest</b> method that can be used
      to determine which part of the grid the event refers to.</li>
    <li>
      <b>Selection</b>:
      The <b>selectionChanging</b> and <b>selectionChanged</b>
      events can be used to prevent users from selecting parts 
      of the grid or to display information relative to the 
      current <b>selection</b>.</li>
    <li>
      <b>Editing</b>: 
      The FlexGrid provides events that can be used for 
      preventing certain cells from being edited 
      (<b>beginningEdit</b>), for customizing the editor
      element (<b>prepareCellForEdit</b>), for
      validation (<b>cellEditEnding</b>), and more.</li>
    <li>
      <b>Resizing Rows and Columns:</b>
      The FlexGrid raises events as rows and columns
      are being resized so you can validate or
      persist the grid layout.</li>
    <li>
      <b>Reordering Columns:</b>
      The FlexGrid raises events as rows and columns
      are being dragged into new positions so you can
      validate or persist the grid layout.</li>
    <li>
      <b>Drag and Drop:</b>
      The FlexGrid uses HTML drag/drop events for reordering
      rows and columns, and you can extend that support to
      provide cell drag and drop functionality as well.</li>
  </ul>  
</div>
